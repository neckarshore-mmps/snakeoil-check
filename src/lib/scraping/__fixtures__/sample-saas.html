How teams use Linear

Linear scales from a 3-person team to enterprise org-wide rollouts. The data model is designed around cycles, projects, and issues — three primitives that compose into any workflow shape.

Each issue has a clear owner, a status, an estimate, and lives in a project. Projects belong to teams. Teams have cycles. The hierarchy is intentional and shallow.

Read the docs at docs.linear.app for the full data model. We also publish the schema as an open GraphQL API at linear.app/graphql.
